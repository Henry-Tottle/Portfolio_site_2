Separation of Concerns

What is Separation of Concerns?

In computer science, separation of concerns (sometimes abbreviated as SoC) is a design principle for separating a computer program into distinct sections. Each section addresses a separate concern, a set of information that affects the code of a computer program. Source - Wikipedia
    So lets apply this to HTML, CSS and JavaScript using a house for reference:
  • HTML could be considered the base structure, such as the bricks, mortar, foundation and roof.
  • CSS could be considered the furnishings and decoration such as the paint on the walls, or the sofa in the living room
  • JavaScript provides the functionality of the page, much like plumbing and electricity.

HTML Syntax

Through frequent use of HTML the syntax becomes almost second nature. I wrote this section of my notes on day one, and I can't say I've actually had to look back at them. That said, it can't hurt to have a bit of info here!

  • HTML is made up of opening and closing tags (mostly!)
  • The things inside the "< >" are considered the tags
  • The things inside the "< >" and "< / >" are considered the elements
  • If you have multiple elements inside other elements it is called nesting (think Russian Dolls)
  • The tag you use will very basically determine where your elements will set relative to one another
  • The property that determines this is whether the element is Block level or Inline
    Block level elements take up the whole line
  • < div >
  • < section >
  • < p >
  • < ul/ol >
  • < form >
    Inline elements go next to each other on the line
  • < span >
  • < a >
  • < img />
  • < li >
  • < br >
    Some useful attributes for tags:
  • href="" creates a link to another website or point on a page, and is mainly used in anchor tags
  • src="" creates a link to a source of an image or an audio clip
  • alt="" creates hidden text for screen readers to view, to increase accessibility for a webpage. This attribute will usually be applied to an image

Semantic Elements

Semantic elements are a useful way for the author of an HTML document to describe elements clearly. The tag used often work the same way as a < div> or a < span> but have a name which will actually describe its intentions.

    Examples of semantic tags are:
  • < header>
  • < nav>
  • < main>
  • < footer>
  • < aside>

Forms

So far, all we know about forms are the various input types you can use and their intention. The user will fill out the form with all the relevant information. Once they press submit, that information will often then be sent to the back end, to be stored in a database. There will probably be some kind of validation process at this stage.

    Some examples of input type are:
  • "button"
  • "checkbox"
  • "email"
  • "password"
  • "url"
  • "number"
